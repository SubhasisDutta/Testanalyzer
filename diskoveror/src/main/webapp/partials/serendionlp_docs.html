URI Parameters Method Description
/api/all text= {string text to be analysed}
file = text file 		GET This API will identify the entities, keywords, topic and Sentiment score for a text and return a JSON or XML response as requested.
/api/all text= {string text to be analysed}
file = text file 		POST This API will identify the entities, keywords, topic and Sentiment score for a text or file and return a JSON or XML response as requested.
/api/entity/all
/api/entity/person
/api/entity/organization
/api/entity/location
/api/entity/currency
/api/entity/percent
/api/entity/date
/api/entity/time		 nlp=stanfordnlp or opennlp (default is Stanford NLP)
text= {string text to be analysed}
file = text file 		GET This API will identify the entities present in the text and return a JSON or XML response as requested.
/api/category/all
/api/category/dbpedia
/api/category/consise		 nlp=stanfordnlp or opennlp (default is Stanford NLP)
text= {string text to be analysed}
file = text file 		GET This API will identify the category a text can belong and list out a detailed list of attributes the text has and can be used for various analysis. The response is JSON or XML.
/getalldetails nlp=stanfordnlp or opennlp (default is Stanford NLP)
text= {string text to be analysed}
file = text file 		GET This returns both entity and category details as JSON and XML as requested.
/getallnlp text= {string text to be analysed}
file = text file		 GET This returns all entity detected by StanfordNLP and OpenNLP package. The result can be used for comparison or further optimization by considering the union or common of the two results.
